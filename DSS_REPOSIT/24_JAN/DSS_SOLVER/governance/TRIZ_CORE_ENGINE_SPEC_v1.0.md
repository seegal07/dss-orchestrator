# TRIZ_CORE_ENGINE_SPEC_v1.0

Базовый документ: DSS_SOLVER/governance/DSS_TRIZ_STRATEGIC_BASELINE_v1.0.md

## 1. Назначение
TRIZ_CORE_ENGINE — обязательный механизм формирования решения в DSS_TRIZ. Решение считается валидным только если получено через TRIZ‑ядро. DSS без TRIZ‑механизма считается INVALID.

## 2. Обязательные элементы TRIZ‑ядра
Каждый элемент обязателен; отсутствие любого блокирует DSS.

1) Противоречие (X/Y, измеримое)
- Формат: improve_x / worsen_y / statement_1l / measurement_hint.
- Требование: X и Y наблюдаемы и измеримы.

2) IKR / IFR
- Формат: краткое описание целевого результата без механизма.
- Требование: IKR не содержит «как» (механизм), только «что» (эффект).

3) Обострение противоречия
- Формат: усиленная формулировка противоречия, показывающая пределы допустимого.
- Требование: обострение усиливает конфликт X/Y, не противоречит ограничениям.

4) Separation (тип + логика)
- Типы: time / space / condition / part‑whole.
- Требование: минимум 1 стратегия разделения, привязанная к противоречию и ограничениям.

5) Principle → Action
- Формат: минимум 2 принципа + конкретные действия изменения системы.
- Требование: каждое действие однозначно выводится из принципа.

## 3. Порядок работы (TRIZ pipeline)
TRIZ‑механика обязательна и выполняется последовательно:
1) Формирование противоречия (X/Y).
2) Формирование IKR/IFR.
3) Обострение противоречия.
4) Формирование стратегии separation.
5) Формирование principle → action.
6) Генерация концептов, где каждый концепт трассируется к separation или principle‑action.
7) Выбор решения и фиксация критериев.

Решение может появиться ТОЛЬКО после шага separation.
Если отсутствует любой элемент TRIZ‑ядра, DSS блокируется.

## 4. Gate‑критерии TRIZ‑ядра (Gate_TRIZ_*)
Новые гейты валидируют именно TRIZ‑механику и исключают «разумные», но не TRIZ‑решения.

### Gate_TRIZ_1 — CONTRADICTION_MEASURABLE
PASS:
- X/Y измеримы и подтверждены measurement_hint.
FAIL:
- X/Y не измеримы или не заданы.
BLOCKED:
- отсутствует противоречие.

### Gate_TRIZ_2 — IKR_DEFINED
PASS:
- IKR задан как эффект без механизма.
FAIL:
- IKR содержит механизм.
BLOCKED:
- IKR отсутствует.

### Gate_TRIZ_3 — CONTRADICTION_SHARPENING
PASS:
- Обострение противоречия задано и усиливает X/Y.
FAIL:
- Обострение не усиливает конфликт или противоречит ограничениям.
BLOCKED:
- Обострение отсутствует.

### Gate_TRIZ_4 — SEPARATION_STRATEGY
PASS:
- Есть минимум 1 стратегия separation с логикой и привязкой к ограничениям.
FAIL:
- Стратегия не связана с X/Y или ограничениями.
BLOCKED:
- Separation отсутствует.

### Gate_TRIZ_5 — PRINCIPLE_ACTION
PASS:
- Есть минимум 2 принципа и для каждого — конкретное действие.
FAIL:
- Принцип без действия или действие не выводится из принципа.
BLOCKED:
- Принципы/действия отсутствуют.

### Gate_TRIZ_6 — TRACEABILITY_TO_CONCEPTS
PASS:
- Каждый концепт явно трассируется к separation или principle‑action.
FAIL:
- Концепт без источника TRIZ.
BLOCKED:
- Концепты отсутствуют.

### Gate_TRIZ_7 — DECISION_FROM_TRIZ
PASS:
- Обоснование решения явно ссылается минимум на 1 separation и 1 principle‑action.
FAIL:
- Решение обосновано только «рационально», без TRIZ‑механики.
BLOCKED:
- Решение отсутствует.

## 5. Правило валидности
Если любой Gate_TRIZ_* = FAIL/BLOCKED → решение INVALID.
TRIZ‑механика — обязательный источник решения.
