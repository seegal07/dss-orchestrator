# DSS Guided Correction — E1_REAL_CASE_001

Цель: закрыть MISSING по decision‑critical зонам. Только вопросы.

## S05 / Gate5 (IKR)
1) Как выглядит конечный результат без механизма: какое состояние системы считается идеальным, если HQ/филиал работают корректно? (1–2 предложения)
2) Какие ограничения не должны быть нарушены в этом идеальном результате? (список)

## S06 / Gate6 (Hard Barrier)
3) Сформулируйте жёсткий барьер: какие два условия взаимоисключаются и не могут выполняться одновременно в текущей системе?
4) Подтвердите, что барьер эквивалентен противоречию из Step4 (да/нет).

## S11 / Gate11 (Secondary Contradiction)
5) Возникает ли новое противоречие после возможных решений? (true/false) Если true — кратко опишите его.

## S12 / Gate12 (Decision)
6) Какой концепт/решение выбирается (индекс + название)?
7) По каким причинам выбран именно он (связь с метриками и ограничениями)?
8) Какие критерии успеха фиксируются для выбранного решения (числа/срок)?

## Дополнительно для формирования решений
9) Сформулируйте 3–5 возможных концептов (Step9), чтобы было из чего выбирать.
10) Для выбранного концепта дайте базовое описание механизма (Step10) — без деталей реализации.
