USER FLOW (Просто)

Цель: превратить «сырой» бизнес‑вопрос в проверенный пакет решения и исполнения.

Принципы:
- Владелец правила по умолчанию: КЛИЕНТ.
- Клиент НЕ видит PASS/FAIL Harness, только интерпретацию.
- FAIL = диагностический статус, а не отказ клиенту.

Шаг 1 — Вход (15–30 мин)
- Клиент описывает проблему и ограничения.
- Выход: ясная формулировка + владелец + ограничения.

Шаг 2 — Структура (30–60 мин)
- Границы, акторы, взаимодействия, противоречие, ресурсы.
- Выход: структурная модель проблемы.

Шаг 3 — Синтез (30–60 мин)
- Выбор формы, набор решений, механизмы.
- Выход: набор решений + механизм исполнения.

Шаг 4 — Decision Lock (обязательный)
- Фиксируется одно решение.
- Фиксируются отвергнутые альтернативы.
- Фиксируется цена ошибки.

Шаг 5 — Решение и план (30–60 мин)
- Финальный выбор, план внедрения, правило.
- Выход: план + операционное правило.

Шаг 6 — Валидация (5–10 мин)
- Harness проверяет Gate0–Gate14.
- Выход (клиент):
  A) «Решение готово»
  B) «Требует обязательного уточнения входных данных»
  C) «Решение невозможно при данных условиях»

Шаг 7 — Передача (10–15 мин)
- Клиент получает отчёт + план + правило.

Итого (ручной режим): ~2–4 часа на кейс.
